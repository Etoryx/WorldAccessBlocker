# 🌍 WorldAccessBlocker

**WorldAccessBlocker** временно закрывает игрокам доступ к **Аду**, **Энду**, **полёту на элитрах** и **любым отдельным мирам** — до заданной даты или по недельному расписанию. Плагин написан под **Paper** и его форки (**Purpur**, **Folia**) и использует родные Adventure- и Folia-планировщики.

Удобно, когда вайп только стартовал и вы хотите открывать Ад через неделю, пускать в Энд только по выходным или держать лобби закрытым в рабочие часы.

---

## ✨ Возможности

- ⏳ **Блокировка по дате** — закрыть возможность до конкретной даты и времени.
- 📅 **Недельное расписание** — открывать доступ только в нужные дни и часы (например, по выходным или в воскресенье с 15:00 до 17:00).
- 🌋 **Блок Ада** — отменяет создание портала, переход через портал и телепорт эндер-жемчугом.
- 🐉 **Блок Энда** — не даёт активировать портал в Энд (и попасть туда).
- 🪽 **Контроль элитр** — можно запретить надевание и/или сам полёт по отдельности.
- 🗺️ **Отдельные миры** — ограничить любой мир по имени, по дате или расписанию.
- 🚪 **Телепорт при блоке** — заблокированного игрока отправит на спавн указанного мира.
- 🎫 **Персональные исключения** — выдать временный доступ через `/wab bypass`, снять — `/wab remove`.
- 🔌 **Поддержка PlaceholderAPI** — статус блокировки и оставшееся время можно вывести в скорборд или TAB.
- 🌐 **Два языка из коробки** — русский и английский; свой язык добавляется отдельным `.yml`.
- ♻️ **Перезагрузка на лету** — `/wabreload` применяет изменения конфига без рестарта сервера.
- 🧵 **Родная поддержка Folia** — задачи идут через региональный планировщик, без предупреждений о потоках.
- 🔔 **Уведомления об обновлениях** — админы видят сообщение в игре, когда вышла новая версия.

## 💻 Совместимость

| Платформа | Версии | Статус |
|---|---|---|
| Paper | 1.20.1+ | ✅ Полная поддержка |
| Purpur | 1.20.1+ | ✅ Полная поддержка |
| Folia | 1.20.5+ | ✅ Родной планировщик |
| Spigot / Bukkit | — | ❌ Не поддерживаются |

**Нужна Java 17+** (Minecraft 1.20.1–1.20.4 работают на Java 17, 1.20.5+ — на Java 21). Плагин собран под Paper 1.20.1 и работает на всех версиях от 1.20.1 до последней. Spigot и Bukkit не подходят: плагин использует Adventure и планировщики Paper.

## 📦 Установка

1. Скачайте свежий `WorldAccessBlocker.jar`.
2. Положите его в папку `plugins/` на сервере.
3. Перезапустите сервер.
4. Отредактируйте `plugins/WorldAccessBlocker/config.yml` под себя и выполните `/wabreload`.

---

## ⚙️ Настройка

```yaml
# Язык сообщений: "ru" или "en"
language: "ru"

# Часовой пояс сервера (важен для недельного расписания!)
time-zone: "Europe/Moscow"

nether:
  disable: true
  # Работает, только если НЕТ блока `recurring` (закрыто до этой даты):
  restriction-date: "2025-03-10 00:00:00"
  disable-portal-creation: true
  disable-teleportation: true

  # Недельное расписание. Если этот блок есть, `restriction-date` ИГНОРИРУЕТСЯ.
  # Каждый период — это ОКНО, когда доступ ОТКРЫТ. Вне окон — закрыт.
  recurring:
    periods:
      # Открыто только по воскресеньям с 15:00 до 17:00
      - days: [SUNDAY]
        start-time: "15:00"
        end-time: "17:00"
      # Открыто весь день по выходным
      - days: [SATURDAY, SUNDAY]

end:
  disable: true
  disable-portal-activation: true
  # Пустой список = нет открытых окон = закрыто ВСЕГДА
  recurring:
    periods: []

elytra:
  disable: true
  disable-equip: true
  disable-flight: true

custom-worlds:
  lobby_world:
    disable: true
    recurring:
      periods:
        - days: [MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY]
          start-time: "09:00"
          end-time: "17:00"

# Куда телепортировать заблокированных (имя мира → его спавн)
fallback-spawns:
  default: "world"
  nether: "hub"
```

> **Как читать расписание:** когда задан `recurring:`, каждый период описывает **открытое** окно. Попадает текущий день и время в период — доступ **открыт**, иначе **закрыт**. Пустой `periods: []` означает **закрыто всегда**.

## 🧩 Команды и права

| Команда | Право | По умолчанию | Что делает |
|---|---|---|---|
| `/wab bypass <игрок> <объект> <секунды>` | `wab.bypass` | op | Выдать временный доступ |
| `/wab remove <игрок> <объект>` | `wab.bypass` | op | Снять исключение |
| `/wab status [игрок]` | `wab.status` | все | Показать статус блокировок |
| `/wabreload` | `wab.reload` | op | Перезагрузить конфиг без рестарта |

`<объект>` — это `nether`, `end`, `elytra` или имя отдельного мира.

Отдельное право `wab.update-notify` (по умолчанию op) отвечает за уведомления о новых версиях в игре.

## 🔧 PlaceholderAPI

| Плейсхолдер | Возвращает |
|---|---|
| `%wab_nether_blocked%` | `true` / `false` |
| `%wab_end_blocked%` | `true` / `false` |
| `%wab_elytra_blocked%` | `true` / `false` |
| `%wab_time_left_nether%` | сколько осталось (или расписание) |
| `%wab_time_left_end%` | сколько осталось (или расписание) |
| `%wab_time_left_elytra%` | сколько осталось (или расписание) |

## ❓ Частые вопросы

**Работает ли на Folia?**
Да. С версии 0.9.0 плагин сам определяет Folia и раскидывает задачи через региональный планировщик, а на Paper/Purpur — через обычный планировщик Bukkit.

**Как выдать игроку постоянный доступ к закрытой возможности?**
Задайте большую длительность исключения: `/wab bypass Ник nether 999999999`.

**Можно закрыть мир по дате, а потом перейти на расписание?**
Да. Если под возможностью есть блок `recurring:`, дата игнорируется. Уберите `recurring:`, чтобы вернуться к режиму по дате, и выполните `/wabreload`.

# 📋 Список изменений — 0.11.0

### Добавлено

- **Поддержка Minecraft 1.20.1 — последняя версия.** Плагин теперь запускается на Paper / Purpur / Folia начиная с **1.20.1** (Java 17+), а не только с 1.20.5. Один и тот же JAR работает и на старых 1.20.x, и на самых свежих версиях игры.
- **Статус блокировок для всех.** Теперь любой игрок может посмотреть, что и до когда закрыто, командой `/wab status` — op больше не нужен. Выдача и снятие исключений (`/wab bypass`, `/wab remove`) остаются только у операторов.
- **Элитры отключаются даже в полёте.** Если игрок уже летит, когда начинается окно блокировки, его теперь тоже опустит на землю (раньше отменялся только взлёт).
- **Предупреждение об устаревшем конфиге.** При старте сервер подскажет в консоли, если ваш `config.yml` старее текущей версии плагина.
- **Мягкий откат языка.** Если в языковом файле не хватает какой-то фразы, плагин подставит её на английском вместо «Message not found».

### Исправлено

- **Folia: больше никаких крашей.** Плагин перестал падать при сохранении исключений и корректно телепортирует игроков между мирами и регионами.
- **Цветные уведомления о новой версии.** Сообщение об обновлении теперь выводится с цветами, а не с символами форматирования в тексте.
- **Выход из Энда.** Покинуть Энд больше нельзя запретить случайно: при закрытом Энде блокируется только вход, выход работает.
- **Перенос старых настроек расписания** при обновлении конфига теперь срабатывает как надо.

### Изменено

- Права проверяются отдельно для каждой подкоманды.
- В сборку релиза добавлен прогон тестов, а список поддерживаемых версий на Modrinth формируется автоматически — все релизы от 1.20 и выше.
